# 🏠 Arenda — сервис аренды квартир посуточно

## 📌 О проекте

**Arenda** — это веб-приложение для поиска и бронирования квартир посуточно. Проект разработан в рамках курсовой работы. Система объединяет арендодателей и арендаторов, предоставляя удобный интерфейс для размещения объявлений, поиска жилья, управления бронированиями и отзывами.

### Основные возможности

#### Для арендодателей:
- Размещение и управление объявлениями о квартирах
- Просмотр бронирований на свои квартиры
- Подтверждение или отмена броней
- Статистика по доходам и загрузке

#### Для арендаторов:
- Поиск квартир с фильтрами (город, даты, цена, количество комнат)
- Просмотр детальной информации о квартире (фото, удобства, отзывы)
- Бронирование квартир на выбранные даты
- Личный кабинет с историей бронирований
- Избранное

#### Для администраторов и модераторов:
- Модерация отзывов и объявлений
- Управление пользователями

## 🚀 Запуск проекта

### 1. Клонирование репозитория
```bash
git clone https://github.com/sofia-afk01/arenda-project.git
cd arenda-project
```

### 2. Настройка базы данных
- Установите PostgreSQL
- Создайте базу данных:
```sql
CREATE DATABASE arendadb;
```
- Выполните скрипты из папки docs/database/ в следующем порядке:
  * `schema.sql` — создание таблиц;
  * `indexes.sql` — создание индексов;
  * `seed.sql` — заполнение справочников и тестовых данных.

## 3. Подключение проекта интегратора (ВАЖНО!)

Проект интегратора (`Arenda.DBeaver`) — это библиотека для работы с базой данных. Без неё бэкенд не соберётся.

### 3.1. Убедитесь, что проект интегратора есть в решении

1. Откройте **Visual Studio**.
2. Откройте решение `Arenda.API.csproj` (лежит в папке `backend/Arenda.API/`).
3. В **Обозревателе решений** проверьте, есть ли проект `Arenda.DBeaver`.

**Если проекта нет:**

1. Нажмите **ПКМ** на названии решения → **«Добавить»** → **«Существующий проект…»**.
2. Найдите файл `Arenda.DBeaver.csproj` (обычно лежит в папке `backend/Libraries/Arenda.DBeaver/`).
3. Выберите его и нажмите **«Открыть»**.

### 3.2. Добавьте ссылку на проект интегратора в `Arenda.API`

1. В **Обозревателе решений** разверните проект `Arenda.API`.
2. Нажмите **ПКМ** на **«Зависимости»** (Dependencies).
3. Выберите **«Добавить ссылку на проект…»**.
4. В открывшемся окне на вкладке **«Проекты»** отметьте галочкой `Arenda.DBeaver`.
5. Нажмите **«ОК»**.

### 3.3. Перестройте решение

1. В меню: **Сборка** → **Перестроить решение** (Rebuild Solution).
2. В нижнем левом углу должна появиться зелёная надпись **«Сборка успешно завершена»**.

> ⚠️ **Если сборка не удалась** — проверьте, что оба проекта (`Arenda.API` и `Arenda.DBeaver`) загружены и нет битых ссылок, а также убедитесь, что у вас скачано ASP.NET Core 8.0.

Ссылка для установки ASP.NET Core 8.0: https://dotnet.microsoft.com/ru-ru/download/dotnet/thank-you/runtime-aspnetcore-8.0.26-windows-x64-installer?cid=getdotnetcore

### 4. Запуск бэкенда
Важно: Загрузка фотографий работает только при ручном запуске через PowerShell. Запуск через Visual Studio может вызвать проблемы с путями к файлам.
Пошаговая инструкция:
1. Откройте PowerShell (Win + R → powershell → Enter)
2. Перейдите в папку с проектом Arenda.API:
```powershell
cd 'E:\3 курс\технология разработки программного обеспечения\курсовая\Arenda\Arenda.API\Arenda.API'
```
> ⚠️ Путь может отличаться в зависимости от того, куда вы сохранили проект. Укажите актуальный путь к файлу "\Arenda.API.csproj" на вашем компьютере.
3. Убедитесь, что строка подключения к базе данных в файле appsettings.json настроена правильно:
```json
"ConnectionStrings": {
  "DefaultConnection": "Host=localhost;Port=5432;Database=arendadb;Username=ваш_логин;Password=ваш_пароль"
}
```
4. Запустите проект командой:
``` powershell
dotnet run --urls "https://localhost:7151/"
```
5. API будет доступен по адресу: https://localhost:7151
Фронтенд настроен на работу именно с этим HTTPS-адресом.
6. Чтобы остановить сервер, нажмите в окне PowerShell сочетание клавиш Ctrl + C.


### 5. Запуск фронтенда
**Шаг 1. Откройте папку с фронтендом в Visual Studio Code**

1. Запустите **Visual Studio Code**.
2. Выберите **Файл → Открыть папку...** (в английской версии: **File → Open Folder...**).
3. Найдите и выберите папку `frontend/arenda` (в ней должен находиться файл `manage.py`).
4. Нажмите **«Выбор папки»** (в английской версии — **Select Folder**).


**Шаг 2. Создайте виртуальное окружение**

```bash
python -m venv env
```

**Шаг 3. Активируйте виртуальное окружение**

 - Windows:
```bash
env\Scripts\activate
```
 - Mac/Linux:
```bash
source env/bin/activate
```

**Шаг 4. Установите зависимости**

```bash
pip install django
```
```bash
pip install requests
```

**Шаг 5. Запустите сервер Django**
```bash
python manage.py runserver
```

> ⚠️ Важно: Фронтенд настроен на работу с бэкендом по адресу https://localhost:7151. Убедитесь, что бэкенд запущен до открытия сайта.

> ❗ Примечание: Если фотографии на сайт подгружаются некорректно - выключите проект и проверьте на следующий день, в связи со сборкой проекта не все пакеты успевают полностью установиться на ваш ПК.

## Устранение типичных проблем

| Проблема | Причина | Решение |
|----------|---------|---------|
| `address already in use` | Порт 7151 занят | остановить проект бэкенда | findstr :7151` → `taskkill /PID <PID> /F` |
| `failed to push some refs` | В репозитории уже есть файлы | `git pull --allow-unrelated-histories` |
| Фото не грузятся | Не настроена статика в .NET | Добавить `UseStaticFiles` для папки `uploads` |
| Ошибка 400 при регистрации | Пароль короче 6 символов | Использовать пароль длиной 6+ символов |
| `ModuleNotFoundError: No module named 'requests'` | Не установлены зависимости | `pip install -r requirements.txt` |



## API Endpoints (основные)

| Метод | Эндпоинт | Описание |
|------|----------|---------|
| POST | `/api/auth/login` | Вход в систему |
| POST | `/api/auth/register-landlord` | Регистрация арендодателя |
| POST | `/api/auth/register-tenant` | Регистрация арендатора |
| GET | `/api/apartments` | Получить все квартиры |
| GET | `/api/apartments/{id}` | Получить квартиру по ID |
| POST | `/api/apartments` | Создать объявление (требуется токен) |
| POST | `/api/bookings` | Создать бронирование |
| GET | `/api/users/{id}/bookings` | Получить бронирования пользователя |
| GET | `/api/apartments/search` | Поиск квартир с фильтрами |

## 📋 Примечание для проверяющего

- Бэкенд запускается на `https://localhost:7151`
- Фронтенд запускается на `http://127.0.0.1:8000`
- Для проверки API используйте Swagger: `https://localhost:7151/swagger`
- Тестовый пользователь создаётся через POST `/api/Auth/register`
- Загрузка фото работает только при запуске через `dotnet run` (не через Visual Studio)

**Структура репозитория:**
- `backend/ArendaAPI/` — бэкенд на C#
- `backend/Libraries/Arenda.DBeaver/` — проект интегратора (доступ к БД)
- `frontend/arenda/` — фронтенд на Django
- `docs/database/` — SQL-скрипты для создания БД
